# 飞客信用卡日报 · 卡片图设计蓝图

> 本文档记录 `card_gen.py` 的设计理念、排版规范、内容策略与后续优化方向，作为长期迭代的指导手册。

---

## 一、核心原则

### 1.1 读者价值第一

卡片的受众是**信用卡爱好者/从业者**，他们在朋友圈/群聊中扫一眼卡片的时间不超过 **3 秒**。每一张卡片必须回答三个问题：

| 问题 | 卡片要素 |
|------|---------|
| **今天有什么值得关注的？** | 统计栏（最高回复/覆盖银行/热点银行） |
| **具体在讨论什么？** | 帖子标题 + 银行标签 |
| **热度够不够？** | 回复数 + 热力条可视化 |

### 1.2 信息密度原则

- 卡片尺寸 **1080×1440（3:4）**，适配手机竖屏阅读
- 内容区应填充 **60%~80%** 的垂直空间，避免底部大片空白
- 6~8 条帖子为单张卡片最佳容量（太多挤不下，太空显松散）

### 1.3 视觉层级原则

从上到下阅读路径：

```
Header（品牌认知）
  ↓
Stats Bar（全局认知 — 今天值不值得看）
  ↓
Post Cards（具体信息 — 逐条扫读）
  ↓
Footer（行动引导 — 关注/分享）
```

## 二、排版规范

### 2.1 字体系统

| 层级 | 字体 | 字号 | 字重 | 用途 |
|------|------|:----:|:----:|------|
| 品牌标题 | `Noto Serif SC` | 26px | 900 | "飞客信用卡日报" |
| 分类名 | `Noto Serif SC` | 14px | 700 | 底部"今日热门"等 |
| 帖子标题 | `Instrument Sans` / PingFang SC | 15px | 600 | 正文 |
| 元信息 | `Instrument Sans` | 12px | 400 | 回复数/作者 |
| 统计数字 | `Instrument Sans` | 18px | 800 | KPI 数字 |
| 银行标签 | `Instrument Sans` | 10px | 600 | 右上角 tag |
| CTA | `Instrument Sans` | 12px | 600 | 底部引导语 |

> `Instrument Sans` 用作英文字体保证数字清晰，中文回退到 `PingFang SC`。
> `Noto Serif SC` 衬线体用于品牌和分类名，与无衬线正文形成对比。

| 帖子卡片间距 | — | 10px |
| 帖子内标题距元信息 | — | 5px |
| 底部 | 14px / 20px | — |

## 三、内容策略

### 3.1 卡片分组逻辑

| 卡片 | 内容 | 生成条件 |
|:----:|------|---------|
| 1️⃣ 今日热门 | 全量按回复数 Top 6 | 永远有 |
| 2️⃣ 农业银行 | 该银行帖子 Top 6 | 农业银行帖子 ≥ 2 |
| 3️⃣ 股份行精选 | 招行/交行/浦发/平安/兴业 合并 Top 6 | 合计 ≥ 2 |
| 4️⃣ 更多讨论 | 剩余银行合并 Top 6 | 有剩余帖子 |
| 5️⃣ 全量速览 | 全部帖子 Top 8 紧凑展示 | 永远有 |

> 分组依据每日数据自动变化。未来可改为配置化。

### 3.2 信息要素优先级

每条帖子在卡片中展示：

```
┌──────────────────────────────────┐
│  [农业银行]                   TAG│  ← 右上角，银行归属
│                                  │
│  XX银行调整积分兑换比例           │  ← 标题，核心信息
│                                  │
│  23 回复 · user123  ████░        │  ← 元信息 + 热力条
└──────────────────────────────────┘
```

| 要素 | 必须 | 原因 |
|------|:----:|------|
| 帖子标题 | ✅ | 核心信息载体 |
| 回复数 | ✅ | 热度判断依据 |
| 银行标签 | ✅ | 读者关注归属 |
| 作者 | ⚠️ | 可选，长标题时可省略 |
| 热力条 | ✅ | 视觉化热度对比 |
| 阅读数 | ❌ | 价值较低，节省空间 |

### 3.3 统计栏设计

统计栏展示 **3 个关键指标**，回答"今天值不值得看"：

```
        18             8           农业
      ─────         ─────        ─────
     最高回复       覆盖银行      最多讨论
```

指标选取原则：
- **可量化**：数字越大越有吸引力
- **能比较**：读者可以跟昨天对比
- **无歧义**：不需要额外解释

### ✅ P1-1 / P1-2 — 已完成

#### 4.1 帖子摘要行 ✅
**解法**：`enrich.py` 调用 LLM 从标题提炼 8~12 字中文摘要。
- 输出：`threads_enriched.json` 含 `summary` + `value_tag`
- `card_gen.py` 自动加载缓存，摘要作主行 + 原标题作副行
- 效果：读者秒懂每帖核心信息

#### 4.2 价值标签 ✅
**解法**：LLM 识别帖子类型，打标签：

| 标签 | 含义 | 色值 | 状态 |
|------|------|:----:|:----:|
| `限时` | 限时活动/优惠 | `#dc2626` | ✅ LLM 分类 |
| `避坑` | 踩坑提醒/风险提示 | `#ea580c` | ✅ |
| `攻略` | 经验分享/技巧 | `#16a34a` | ✅ |
| `实测` | 亲测结果/数据 | `#2563eb` | ✅ |
| `公告` | 银行官方公告 | `#78716c` | ✅ |
| `讨论` | 一般讨论 | `#94a3b8` | ✅ 默认值 |

**运行流程**：
```bash
python enrich.py          # LLM 摘要+标签，自动增量
python card_gen.py        # 读取缓存生成卡片
```

## 六、技术约束

### 6.1 渲染引擎
- 使用 **Playwright + Chromium** 截图
- 像素颗粒度：1x（无 Retina），保持文件体积
- Viewport 尺寸 = 卡片尺寸

### 6.2 数据流
```
threads_raw.json         ← 抓取
       ↓
threads_filtered.json   ← 分类（含 category 字段）
       ↓
card_gen.py              ← 分组 + 渲染
       ↓
_cards/card_*.png       ← 输出
```

### 6.3 环境兼容
- Windows 控制台 GBK 编码问题：所有 emoji 用文字替代
- Google Fonts 需外网访问，Playwright 渲染时确保网络可用
- 每个卡片的临时 HTML 在截图后自动删除
