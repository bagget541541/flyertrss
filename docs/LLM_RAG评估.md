# 卡片优化：LLM+RAG 价值评估

## 现状摘要

| 卡片类型 | 当前编辑总结 | 各帖结论/建议 |
|---|---|---|
| info_card (信息图) | 模板文字，如「关于xxx的N条讨论中…」、「涉及限时活动…」 | 只有一条编辑总结 |
| top3_card (前三甲) | 同上模板 | 同一条编辑总结 |
| card_01~04 (列表卡) | 无 | 无，只有标题+回复数 |

---

## 评估：LLM+RAG 方案

### 架构设计

```
                                 ┌─────────────────────┐
标题 + 分类 + 回复数 ──────────→ │  LLM (大模型)        │
                                 │                     │
热评内容 (RAG context) ────────→ │  分析帖子的实质内容   │
                                 │  → 给出观点倾向       │
帖子标题中提取关键信息 ────────→ │  → 给出行动建议       │
                                 └─────────┬───────────┘
                                           ▼
                              {
                                "opinion": "农行这个比例调整后，高端卡用户影响不大，但普通用户建议月底前用完里程",
                                "action": "建议：6月底前完成兑换，之后比例将缩水50%",
                                "confidence": "high",
                                "risk_flag": false
                              }
```

### RAG 的上下文来源

| 来源 | 说明 | 可用性 |
|---|---|---|
| 帖子标题 | 已有关键信息 | ✅ 已有 |
| 帖子分类 | 银行类别/话题类型 | ✅ 已有 |
| 社区热评 | 通过 playwright 抓取的 1-4 条热评 | ⚠️ 非所有帖子都有 |
| 历史同类帖子 | 相同分类的历史帖子统计 | ❌ 需要建索引 |
| 银行政策背景 | 外部知识（如央行政策、各行信用卡权益表） | ❌ 需要建知识库 |

### 价值评估

#### 👍 高价值场景

| 场景 | 说明 | LLM+RAG 提升效果 |
|---|---|---|
| **权益变更帖** | "农行里程比例调整" | 模板文字只能写"涉及权益变化"，LLM+RAG 能给出「比例从x:1变为y:1，高端卡不受影响」的具体分析 |
| **对比选择帖** | "科创白vs黑白菜怎么选" | 当前看选项关键词生硬拼接，LLM+RAG 能基于热评总结两卡优劣势 |
| **避坑预警帖** | "某某套路提醒" | 当前写"曝光了一个风险点"，LLM+RAG 能写出具体操作风险环节 |
| **活动攻略帖** | "XX银行满减" | 当前写"有参考价值"，LLM+RAG 能算出最佳姿势 |

#### 👎 低价值场景

| 场景 | 说明 | 原因 |
|---|---|---|
| **普通求助帖** | "请问XX卡怎么样" | 回复多为个人观点，LLM 也难总结出可靠结论 |
| **资讯分享帖** | "某银行发了新卡" | 只需告知存在即可，不需要 LLM 分析 |
| **低回复帖** | 回复 < 5 条 | 信息量不足以支撑 RAG 分析 |

#### 量化收益预估

```
每条帖子 LLM 成本：~0.02 元（按市面通用模型）
每日帖子数：10-20 条
每日 LLM 成本：0.2-0.4 元
每日抓取热评时间（playwright）：~30s/帖 → 5-10 分钟

vs 人工价值：一个编辑做这些分析要 20-30 分钟/天
→ ROI 非常高，日均成本不到 5 毛钱
```

### 实施建议

#### 分阶段推进

**Phase 1 — 仅 top3 + 信息图卡（5 条/天）**
- 成本最低：~0.1 元/天
- 覆盖最受关注的核心帖子
- RAG context = 标题 + 分类 + 已有热评
- Prompt 输出 JSON：`{opinion, action_tip, confidence}`

**Phase 2 — 扩展至回复 > 10 的帖子**
- 约 8-12 条/天
- 成本 ~0.2 元/天
- 需要 `enrich.py` 批量处理，缓存结果到 `threads_enriched.json`

**Phase 3 — 全量 + 知识库**
- 所有帖子
- 引入历史知识库（银行权益库、历史分类数据）
- 需要建向量索引（chroma/lance 本地轻量方案）

#### Prompt 设计（Phase 1 示例）

```
你是信用卡分析师。分析这条帖子，给出鲜明观点和行动建议。

帖子标题：{title}
帖子分类：{category}
社区热评：
{hot_replies_joined}

请返回 JSON：
{
  "opinion": "你的观点倾向，必须有明确立场（看好/保留/建议避开）",
  "action_tip": "给读者的具体行动建议，什么时间做什么事",
  "confidence": "high/medium/low"
}
```

#### 卡片渲染变化

编辑总结区从：
```
编辑总结
「关于科创白和黑白菜怎么选」的23条讨论中观点分化明显…
具体截止时间请以官方公告为准。
```
变成：
```
编辑点评 👍
科创白权益扎实但门槛高，黑白菜年费灵活但境外权益缩水。
→ 长期用卡选科创白，短期过渡可考虑黑白菜；建议赶在6月底活动结束前申卡。
```

---

## 结论

| 项目 | LLM+RAG 价值 | 优先级 |
|---|---|---|
| **去帖子作者** | — | ✅ 已实现 |
| **编辑总结更有观点** | 🟢 **价值高** | **P0 — 下一做** |
| **每帖结论/行动建议** | 🟢 价值高，但需分阶段 | P1 — Phase 1 先 core 5 帖 |

**一句话：** 值得做。日均成本不到 5 毛，产出是模板文字的质变。建议 Phase 1 先上线 top3 + 信息图卡的 LLM+RAG 点评，跑一周看效果再扩展。
