# 公众号图文优化清单

> 面向实际公众号发表，以读者价值和吸引力为导向梳理。
> 优先级：直接影响打开率和阅读量 > 提升阅读体验 > 锦上添花

---

## P0 — 直接影响打开率

### 1. 文章标题重写 ✅

- **现状**：直接使用论坛原标题（口语化、信息密度低）
- **目标**：LLM 重写为公众号风格标题
- **要求**：
  - 包含数字（如"3 张卡"、"6 月活动"）
  - 痛点或利益点前置（如"别再选错了"、"省钱攻略"）
  - 控制在 22 字以内（公众号标题截断线）
- **改动**：`enrich.py` 增加 `wechat_title` 字段生成
- **进度**：
  - 2026-06-04: 新增 `wechat_title` 字段生成（`enrich.py` LLM prompt）
  - 2026-06-04: 新增 `_smart_truncate` 智能截断
  - **2026-06-04 v2**: 修复 LLM 未返回 wechat_title 的问题（prompt 加入显式数组示例 + MUST NOT be empty 约束）
  - **2026-06-04 v2**: 新增 `_make_wechat_fallback()` 兜底逻辑，LLM 返回空时从标题自动截取
  - **2026-06-04 v2**: 新增文章级标题 `article_title` + 摘要 `article_desc` 自动生成

### 2. 封面图主标题优化 ✅

- **现状**：`template-cover.html` 主标题用 10 字摘要，副标题用原标题
- **问题**：摘要太短、信息量不足，封面在信息流中不够醒目
- **目标**：
  - 主标题用 LLM 重写的公众号标题（大字、醒目）
  - 副标题放关键数据（"65 条讨论 · 8 家银行"）
  - 保留价值标签色块（限时/避坑等）
- **改动**：`card_gen.py::render_cover()` + `template-cover.html`
- **完成**：2026-06-04 实现
  - `enrich.py` 新增 `wechat_title` 字段生成
  - `card_gen.py::render_cover()` 优先使用 `wechat_title`，fallback 到 `summary`
  - `template-cover.html` 副标题改为 `{subtitle}`（数据化）

### 3. 第一卡（今日热门）加钩子 ✅

- **现状**：第一条帖子标题直接展示，无引导语
- **目标**：卡片顶部增加"今日最值得关注"引导文案
- **改动**：`template.html` header 区域或 `card_gen.py` 渲染逻辑
- **完成**：2026-06-04 实现
  - `template.html` 新增 `.hook` 样式 + `{hook}` 占位符
  - `card_gen.py` 5 条板块定制钩子文案（"今日社区最热的 N 条讨论"等）
  - 渲染测试覆盖

### 4. 编辑点评调优 ✅

- **现状**：LLM prompt 已加强，但 fallback 模板仍偏泛化
- **目标**：
  - 每条点评必须引用 1 条具体热评内容
  - 行动建议要具体到时间节点
  - 避免"建议关注"等空话
- **改动**：持续调优 `_gen_llm_opinion` prompt + `_gen_editor_note` 模板
- **完成**：2026-06-04 实现
  - `_gen_llm_opinion` prompt 增强：引用热评观点 + 具体时间节点 + 禁止空话
  - `_gen_editor_note` 模板数据化：引用回复数 + 当前日期
  - 函数签名简化：`_gen_editor_note(post)` 移除冗余参数

## P1 — 提升阅读体验

### 5. 卡片数量精简 ✅

- **现状**：5-6 张分类卡 + top3 + info + 封面 = 8-9 张图
- **问题**：公众号文章配图太多反而劝退，读者注意力分散
- **目标**：3-5 张图（1 封面 + 2-3 卡片 + 1 info 或 top3）
- **方案**：
  - 合并"农业银行"和"更多讨论"为一张卡
  - "全量速览"卡改为可选（帖子少时跳过）
  - 或按需动态生成卡片数量
- **改动**：`card_gen.py::main()` 分组逻辑
- **完成**：2026-06-04 实现
  - 农行/股份行/其他合并为"分类精选"（≤5 条）
  - 全量速览改为 threads > 10 时才出
  - 热门卡 top 6 → top 5
  - 卡片总数从 7-9 张降至 4-6 张

### 6. 信息图卡片（info_card）兜底 ✅

- **现状**：热评为空时显示"暂无高赞评论"，卡片下半空白
- **目标**：热评为空时改用数据亮点填充
  - 回复数 vs 平均值对比
  - 浏览数/回复数比（互动率）
  - 该帖与今日其他帖的热度对比
- **改动**：`card_gen.py::_render_info_card()` + `template-info.html`
- **完成**：2026-06-04 实现
  - 热评空时显示数据亮点卡（回复数/浏览量/互动率）
  - `all_posts_meta` 新增 avg_replies/avg_views/avg_engagement
  - `template-info.html` 新增 `.dh-item` 样式

### 7. Top3 卡片热评截断 ✅

- **现状**：热评截断 120 字，经常断在句中
- **目标**：智能截断（按句号/问号截断）或限制到完整句子
- **改动**：`card_gen.py::fetch_hot_replies_list()` 截断逻辑
- **完成**：2026-06-04 实现
  - 新增 `_smart_truncate(text, max_len)` 函数，按 `。！？…` 截断
  - 替换所有硬编码截断（`[:180]`/`[:150]`/`[:120]`/`[:200]`）
  - 7 个单元测试覆盖各类边界情况

### 8. 价值标签视觉强化 ✅

- **现状**：右上角 10px 小标签，手机端几乎看不清
- **目标**：
  - 标签放大到 12-14px
  - 或改为标题前的色块标记（如 `[限时]` 红色前缀）
- **改动**：`template.html` tag 样式 + `card_gen.py` 渲染逻辑
- **完成**：2026-06-04 实现
  - CSS `font-size: 10px → 13px`，`padding: 1px 8px → 2px 10px`，`max-width: 56px → 72px`
  - `card_gen.py` 内联样式同步更新

## P2 — 锦上添花

### 9. 卡片底部 CTA ✅

- **目标**：统一引导关注，可加公众号二维码占位
- **改动**：`template.html` footer 区域
- **完成**：模板已有 CTA（"关注 {branding} 每日获取信用卡情报"），2026-06-04 确认无需额外改动

### 10. 预览图优化 ✅

- **现状**：`preview.jpg` 是所有卡片缩略图拼接
- **目标**：精选 3 张最有吸引力的卡片拼图（封面 + info + top3）
- **改动**：`card_gen.py::gen_preview()`
- **完成**：2026-06-04 实现
  - 选图优先级：`cover_wechat.png` → `card_top3.png` → 最大编号 info card → 其余
  - 输出为 3×1 横排拼图（720×320）

### 11. 文章摘要自动生成 ✅

- **目标**：生成公众号文章的 description 字段（显示在信息流中）
- **内容**：今日关键数据 + 最热帖标题
- **改动**：`enrich.py` 新增 `article_desc` 字段 + `wechat_article_gen.py` 组装文章元数据
- **完成**：2026-06-04 实现
  - `enrich.py` 自动生成 `article_desc`（"今日3家银行22条讨论，最热帖25条回复"）
  - `wechat_article_gen.py` 输出 `公众号元数据_{ds}.json` 含完整 meta
  - 封面图副标题优先使用 `article_desc`

### 12. 发布时间优化 ✅

- **目标**：早报 7:30-8:00 / 晚报 20:00-21:00 发布
- **改动**：`run.py` + `enrich.py` 支持 `--edition` 参数
- **完成**：2026-06-04 实现
  - `run.py` 新增 `--edition {早报,晚报}` 参数，默认 12:00 前=早报，12:00 后=晚报
  - `enrich.py` 新增 `--edition` 参数，写入 `article.edition` 字段
  - `run.py` Step 2 传递 edition 给 enrich.py
  - `wechat_article_gen.py` 自动从 JSON 读取 edition，无需额外传参
