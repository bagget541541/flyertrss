# 公众号读者价值与引流优化计划

> 面向已完成的 12 项图文优化（P0-P2），从**读者价值**、**信息密度**、**引流转化**三个维度规划下一阶段。
> 核心指标：打开率 → 阅读完成率 → 分享率 → 关注转化率

---

## 一、现状评估

当前文章结构：
```
封面图（品牌+摘要+数据）
→ 今日概览（文字）
→ 前三甲列表+卡片图
→ 分类精选卡片图集
→ 精华预览图
→ CTA（"关注飞客信用卡日报"）
```

**已解决问题**（前 12 项）：标题重写、封面优化、卡片精简、热评截断、编辑点评等。

**仍存在的问题**：

| 问题 | 影响 | 严重度 |
|------|------|:------:|
| 正文几乎纯图片，无文字信息 | 加载慢、信息密度低、无法复制/搜索 | 🔴 |
| 封面标题用 10 字 summary，吸引力不足 | 打开率没充分利用标题优化成果 | 🔴 |
| 编辑点评只出现在卡片图上，正文看不到 | 高价值内容被埋没 | 🟠 |
| CTA 单一，无多级引导 | 关注转化率低 | 🟠 |
| 无互动引导 | 无评论区互动，影响推荐权重 | 🟠 |
| 无引流钩子 | 看完即走，无法沉淀用户 | 🟡 |
| 热度/避坑内容未突出展示 | 好内容被平铺淹没 | 🟡 |
| 文章结构无节奏变化 | 阅读疲劳 | 🟡 |

## P0 — 提升读者价值（直接影响打开率 + 阅读完成率）

### 1. 正文文字化 — 把 LLM 内容写进正文

**目标**：不再只有卡片图，每条帖子在正文中有文字摘要 + 编辑点评

**现状**：`wechat_article_gen.py` 只贴图，`enrich.py` 生成的 `summary`/`wechat_title`/`value_tag` 和 `_gen_llm_opinion` 生成的编辑点评只在卡片图上可见

**方案**：在文章 body 中，每张卡片图前面插入文字摘要区：

```
🏆 前三甲详情
  ┌─ #1 [避坑] 警惕！1场大雨毁掉黑金梦 ─────────────┐
  │  摘要：黑金卡申请失败因大雨，年收入差2万也不行   │
  │  编辑：收入证明看近12个月，差一毛都不行。有卡友  │
  │        反馈"有的地方看上年个税截图"，建议先确认  │
  │        当地政策再申请。                          │
  │  热度：20 回复 · 12091 浏览                      │
  └──────────────────────────────────────────────────┘
  [card_top3.png]  ← 详细图文卡片
```

**改动范围**：
- `wechat_article_gen.py`：生成每帖的文字摘要 + 编辑点评段落
- 使用 `threads_enriched.json` 中已有的 `summary`/`value_tag` 和 `editor_note`（需要将 `_gen_llm_opinion` 输出也存到 enriched 中）

**效果**：
- 读者只看文字就能获取核心信息，图片仅作为视觉补充
- 文字可被微信搜索索引
- 加载速度提升（文字先渲染，图片后加载）

### 2. 封面标题改用 wechat_title

**目标**：封面图主标题从 10 字 summary 改为 22 字以内的公众号标题

**现状**：`template-cover.html` 的 `.content .title` 用的是 `{summary}`（10字）

**分析**：前 12 项优化中"封面主标题优化"只优化了封面调用侧的 `render_cover()` 支持 `wechat_title`，但实际渲染时 cover_title 来自 `article_title`（"飞客晚报 | xxx"），fallback 后才用 summary。但 `template-cover.html` 中 `{summary}` 仍然是 10 字摘要。

**方案**：
- `template-cover.html` 的 `{summary}` 改为 `{wechat_title}`（LLM 重写后的公众号风格标题）
- `card_gen.py::render_cover()` 传递 `wechat_title` 给模板
- 同时增加副标题的信息密度

### 3. 编辑点评同步到正文

**目标**：LLM 生成的编辑点评不仅出现在卡片图上，也出现在文章正文中

**现状**：`_gen_llm_opinion()` 在 `card_gen.py` 中调用，结果只渲染到卡片图（template-info.html 和 template-top3.html），`wechat_article_gen.py` 无法获取

**方案**：
- 将 LLM 编辑点评结果存入 `threads_enriched.json` 每个帖子的 `editor_note` / `editor_action` 字段
- 可在 `enrich.py` 中增加批量调用，或 `card_gen.py` 回写结果
- `wechat_article_gen.py` 读取后插入正文

## P1 — 提升信息密度与阅读体验

### 4. 避坑/限时专区

**目标**：将有高价值标签（避坑/限时）的帖子独立展示，让读者一眼看到"今天要注意什么"

**方案**：
```
⚠️ 今日避坑提醒
  ┌─────────────────────────────────┐
  │  █ 黑金梦碎！一场大雨毁梦      │
  │    收入看近12个月，差2万也不行  │
  │    热度：20 回复                │
  ├─────────────────────────────────┤
  │  █ 运通行悠白温暖？26条回复热议 │
  │    权益调整详情待确认           │
  │    热度：26 回复                │
  └─────────────────────────────────┘
```

**改动范围**：
- `wechat_article_gen.py`：从 posts 中筛选 `value_tag` 为"限时"或"避坑"的帖子
- 生成专属区块，放在文章前部

### 5. 编辑推荐板块

**目标**：文章顶部增加"今日编辑推荐"，给读者一个明确的阅读方向

**方案**：
```
✨ 今日编辑精选
今日最值得关注的是「黑金梦碎」—— 年收入差2万就是不行，
硬性规定50万，建议先确认当地政策再申请。
另外「运通行悠白」26条回复热议中，权益可能有变。
```

**改动范围**：
- `wechat_article_gen.py`：选择 top1 帖子的编辑点评作为推荐语
- 从 `threads_enriched.json` 中取 `editor_note`（需先实现 #3）

### 6. 文章阅读节奏优化

**目标**：图片穿插文字说明，避免连续的图片墙导致阅读疲劳

**现状**：正文 = 封面图 → 概览文字 → top3图 → 卡片图×N → 预览图，中间除概览外无文字

**方案**：
```
封面图
→ 编辑精选（文字）
→ 避坑提醒（文字+小图）
→ 今日热门卡（图+文字说明）
→ 前三甲详情（文字摘要 → top3图）
→ 分类精选（卡片图）
→ 互动引导（文字）
→ CTA（文字）
```

**改动范围**：
- `wechat_article_gen.py`：重新编排 body 顺序，增加文字块

## P2 — 引流转化

### 7. 多段式 CTA 升级

**目标**：从 1 句 CTA 升级为 **3 段式关注引导**

**方案**：
```
━━━ 文章结尾 ━━━

📬 还没看够？
关注「飞客信用卡日报」每日获取：
• 银行活动第一时间推送
• 申卡/提额实战攻略  
• 避坑提醒避免踩坑

💬 你怎么看？
你觉得哪张卡最值得办？评论区聊聊

🔖 回复关键词
关注后回复「黑金」获取完整攻略图
```

**改动范围**：
- `wechat_article_gen.py`：CTA 区域改为多段 HTML
- 关键词回复从当前最热帖的标签/摘要自动生成

### 8. 互动引导

**目标**：引导读者评论、点在看、分享

**方案**：
- 文章末尾增加投票/提问（"今天哪条讨论最有价值？"）
- "转发给需要的朋友"引导
- 评论区话题引导

### 9. 品牌故事植入

**目标**：让读者知道"谁在写"、"为什么值得信任"

**方案**：
- 文章开头或结尾增加编辑简介（专业背景、做日报的初衷）
- 个人品牌：@moat成长 的定位介绍
- 价值承诺："每天 5 分钟，掌握信用卡圈最新动态"

### 10. 公众号二维码 + 社群引流

**目标**：将读者从文章沉淀到公众号/社群/个人号

**方案**：
- 公众号文章末尾放二维码图片（需手动准备素材）
- 或引导至知识星球/交流群
