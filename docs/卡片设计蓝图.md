# 飞客信用卡日报 · 卡片图设计蓝图

> 本文档记录 `card_gen.py` 的设计理念、排版规范、内容策略与后续优化方向，作为长期迭代的指导手册。

---

## 一、核心原则

### 1.1 读者价值第一

卡片的受众是**信用卡爱好者/从业者**，他们在朋友圈/群聊中扫一眼卡片的时间不超过 **3 秒**。每一张卡片必须回答三个问题：

| 问题 | 卡片要素 |
|------|---------|
| **今天有什么值得关注的？** | 统计栏（最高回复/覆盖银行/热点银行） |
| **具体在讨论什么？** | 帖子标题 + 银行标签 |
| **热度够不够？** | 回复数 + 热力条可视化 |

### 1.2 信息密度原则

- 卡片尺寸 **1080×1440（3:4）**，适配手机竖屏阅读
- 内容区应填充 **60%~80%** 的垂直空间，避免底部大片空白
- 6~8 条帖子为单张卡片最佳容量（太多挤不下，太空显松散）

### 1.3 视觉层级原则

从上到下阅读路径：

```
Header（品牌认知）
  ↓
Stats Bar（全局认知 — 今天值不值得看）
  ↓
Post Cards（具体信息 — 逐条扫读）
  ↓
Footer（行动引导 — 关注/分享）
```

## 三、内容策略

### 3.1 卡片分组逻辑

| 卡片 | 内容 | 生成条件 |
|:----:|------|---------|
| 1️⃣ 今日热门 | 全量按回复数 Top 6 | 永远有 |
| 2️⃣ 农业银行 | 该银行帖子 Top 6 | 农业银行帖子 ≥ 2 |
| 3️⃣ 股份行精选 | 招行/交行/浦发/平安/兴业 合并 Top 6 | 合计 ≥ 2 |
| 4️⃣ 更多讨论 | 剩余银行合并 Top 6 | 有剩余帖子 |
| 5️⃣ 全量速览 | 全部帖子 Top 8 紧凑展示 | 永远有 |

> 分组依据每日数据自动变化。未来可改为配置化。

### 3.2 信息要素优先级

每条帖子在卡片中展示：

```
┌──────────────────────────────────┐
│                                  │
│  XX银行调整积分兑换比例           │  ← 标题/摘要，核心信息
│                                  │
│  23 回复  ████░                  │  ← 回复数 + 热力条
└──────────────────────────────────┘
```

| 要素 | 必须 | 原因 |
|------|:----:|------|
| 帖子标题/摘要 | ✅ | 核心信息载体 |
| 回复数 | ✅ | 热度判断依据 |
| 热力条 | ✅ | 视觉化热度对比 |
| 作者 | ❌ | 已移除，降低信息噪声 |
| 银行标签 | ❌ | 信息图卡已移除，普通卡保留 |
| 阅读数 | ❌ | 价值较低，节省空间 |

### 3.3 统计栏设计

统计栏展示 **3 个关键指标**，回答"今天值不值得看"：

```
        18             8           农业
      ─────         ─────        ─────
     最高回复       覆盖银行      最多讨论
```

指标选取原则：
- **可量化**：数字越大越有吸引力
- **能比较**：读者可以跟昨天对比
- **无歧义**：不需要额外解释

**运行流程**：

### 7.4 硬规则（Hard Rules）

以下规则在 `template.html` 和 `card_gen.py` 中共同生效：

1. **卡宽 / 高固定** `1080 × 1440 px`（3:4 竖屏），不支持变尺寸
2. **最小可读字号** `10px`（标签），正文不低于 `12px`
3. **禁止 emoji** — 所有图示使用文字替代（如 🔥 → `[HOT]`），避免 Windows GBK 编码问题
4. **内容密度** `60%~80%` — 超出时截断，不足时补充
5. **单卡片最大帖子数**：普通 6 条，紧凑模式（速览）8 条
6. **禁止假数据** — `replies` 必须来自真实抓取

### 7.5 Token 使用示例

在 `template.html` 中使用时（未来可选 CSS 变量化）：

```css
/* 当前：硬编码 */
.post .title { font-size:15px; font-weight:600; color:#0f172a; }

/* 未来：CSS 变量化 */
.post .title {
  font-size: var(--fs-title);
  font-weight: var(--fw-title);
  color: var(--text-title);
}
```

目前 Token 值直接由 `card_gen.py` 传入 `.format()` 或写在 `template.html` 中硬编码。如需全变量化，将 Token 定义从本文档同步到 `template.html` 的 `:root{}` 即可。
