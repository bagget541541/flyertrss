## 二、数据流追踪：高价值文本去哪了？

```
enrich.py
  ├→ summary: "信用卡经典笑话分享会"         → 只在卡片图上
  ├→ wechat_title: "99%人不知道的信用卡笑话" → 封面图 + 文章标题
  ├→ value_tag: "讨论" / "避坑" / "限时"     → 只在卡片图上
  └→ article_title/desc                       → ✅ 文章标题+摘要

card_gen.py
  ├→ _gen_editor_note(post)                  → 只在卡片图上(info/top3)
  ├→ _gen_llm_opinion(post, hot_replies)     → 只在卡片图上(info/top3)
  └→ 卡片截图 → card_*.png, card_top3.png    → 图片文件

wechat_article_gen.py
  ├→ 文章标题                                  → ✅ 
  ├→ 今日概览文字                              → ✅ 唯一正文文字
  ├→ top3 文字列表（标题+回复数）               → ✅ 
  ├→ 嵌入卡片图                                → 图片
  └→ CTA                                       → ✅ 
```

**核心问题**：

| LLM 产出的文本 | 去哪了 | 读者能看到吗 |
|---------------|--------|:----------:|
| `summary`（10字摘要） | 卡片图内小字 | ❌ 需放大图片 |
| `wechat_title`（22字标题） | 封面图 | ⚠️ 仅封面可见 |
| `value_tag`（限时/避坑等） | 卡片图右上角标签 | ❌ 需放大图片 |
| `editor_note`（编辑点评） | info/top3 卡片图 | ❌ 需放大图片 |
| `editor_action`（行动建议） | info/top3 卡片图 | ❌ 需放大图片 |

**结论**：LLM 花了 80s 生成的高质量文本，99% 被"封印"在图片里，正文完全不展示。

## 三、效果示意：改造前后对比

### 改造前（现状）

> 读者打开文章 → 看到封面图（摘要10字）→ 看到概览文字 → 看到 top3 列表 → 
> 然后连续 5~7 张大图 → 每张图要点击放大 → 才能看到"信用卡经典笑话分享会"这样的摘要 →

| 维度 | 改造前 | 改造后 |

| 帖子摘要 | 卡片图内小字，需放大 | 正文文字，秒读 |
| 编辑点评 | 卡片图内小字，需放大 | 正文文字，可复制/搜索 |
| 价值标签 | 仅卡片图上色块 | 正文中 ⚠️/🔥/🚫 图标+文字 |
| 图片数量 | 7~8 张大图 | 3~4 张（封面+top3+1~2分类） |
| 加载体验 | 所有图同时加载，慢 | 文字先渲染，图片延后加载 |
| 搜索可见性 | ❌ 图片文字不可搜索 | ✅ 正文文字可被微信搜索索引 |

## 四、需要改什么

### `wechat_article_gen.py` — 核心改动

遍历 `posts` 数组，为每条帖子生成文字摘要块：

```python
# 伪代码示意
for p in posts:
    tag = p["value_tag"]
    icon = {"避坑": "🚫", "限时": "⏰", "攻略": "📖", "公告": "📢", "实测": "🔬", "讨论": "💬"}
    summary = p.get("summary", "")
    wechat_title = p.get("wechat_title", "")
    replies = p.get("replies", "?")
    editor_note = p.get("editor_note", "")   # ← 需要从 card_gen.py 同步过来
    
    body += f'''
    <div class="post-text">
      <div class="post-title">{icon[tag]} {wechat_title or summary}</div>
      <div class="post-meta">{tag} · {replies} 回复</div>
      {f'<div class="post-summary">{summary}</div>' if summary else ''}
      {f'<div class="post-editor">📝 {editor_note}</div>' if editor_note else ''}
    </div>
    '''
```

### `enrich.py` 或 `card_gen.py` — 编辑点评回写

当前 `_gen_llm_opinion()` 结果只在生成卡片时使用，需要回写到 `threads_enriched.json`：

```python
# card_gen.py 中，生成 top3_data 时同时回写
for i, p in enumerate(top3_posts):
    opinion, action_tip = _gen_llm_opinion(p, rag_text, post_content=main_content)
    # 回写到 enriched 缓存
    enriched_entry = cmap.get(p["tid"], {})
    enriched_entry["editor_note"] = opinion
    enriched_entry["editor_action"] = action_tip
```
