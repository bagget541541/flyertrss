## 二、路由校正

原方案中的路由路径有误，RSSHub 实际源码如下：

| 项目 | 原方案 | 实际 |
|------|--------|------|
| 域名 | `flyert.com` | `www.flyert.com.cn` |
| 路由 | `/flyert/creditcard/creditcard` | `/flyert/creditcard/:bank` |
| 源页面 | RSSHub 输出 | **直接抓取 `forum-creditcard-1.html`** |
| 页面编码 | UTF-8 | **GBK**（需转码） |
| 链接格式 | `thread-123-1-1.html` | `forum.php?mod=viewthread&tid=123` |

正确的路由参数 `:bank` 对应：
- `creditcard` → 国内信用卡
- `pufa` / `zhaoshang` / `zhongxin` / `jiaotong` / `zhonghang` ... → 各银行子版块

## 三、推荐的落地路径

### 方案 A（推荐）：去掉 RSSHub，直接爬 flyert.com.cn

```
flyert.com.cn → httpx(GBK) → BeautifulSoup解析 → LLM分类 → 日报.md
```

**优点**：
- 无需自建 RSSHub 服务
- 不依赖任何第三方 RSS 中转
- RSSHub 源码中已有完整的解析逻辑可参考
- 延迟更低（少一层代理）

**需要处理**：
1. GBK 编码 → Python `iconv` / `chardet` 解码
2. HTML 解析 → RSSHub 源码中 `util.ProcessFeed()` 可直接参考
3. 反爬策略 → MVP 阶段只解析列表页，不爬详情；请求间隔 3-5s

### 方案 B：Docker 自建 RSSHub（偏重）

```
docker run -d -p 1200:1200 diygod/rsshub
→ http://localhost:1200/flyert/creditcard/creditcard
```

**优点**：跟原方案架构一致，代码改动最小
**缺点**：需要装 Docker，占用约 300MB 内存，MVP 阶段过重

### 方案 C：用 Python feedparser 解析 rsshub.app（不推荐）

Cloudflare 拦截无法绕过，命令行工具做不到 JS 挑战。

## 四、技术细节备忘录

### 4.1 页面结构（关键）

帖子在 HTML 中的结构：

```html
<tbody id="normalthread_4848973">
  <a href="forum.php?mod=viewthread&tid=4848973">
    拿到0072卡了，发现中行复刻了BOCHK经典操作
  </a>
  <cite>作者: Evan14393</cite>
  <em>时间: 7秒前</em>
</tbody>
```

每个 `tbody[id^="normalthread_"]` 代表一条帖子。

### 4.2 分类标签

部分帖子带分类标签（在标题前的 `<em>` 中）：
- `用卡经验`、`中国银行`、`华夏卡` 等
- 可在解析时提前过滤，减少 LLM 调用成本

### 4.3 代理设置

当前环境必须走代理（v2rayN `127.0.0.1:10808`）才能访问 `flyert.com.cn`。
Python 脚本中需配置 `proxies={"http": "http://127.0.0.1:10808", "https": "http://127.0.0.1:10808"}`。

### 4.4 频率控制

RSSHub 源码中对 flyert 标注了 `antiCrawler: false`（无反爬），但仍建议：
- 列表页：每 3-5 分钟爬一次（Discuz 页面有缓存，太快也拿不到新内容）
- 详情页：MVP 阶段**不爬**，只解析列表摘要
- 每日运行 1-2 次（早晚各一）即可满足日报需求
